## 1. 概述

**RB-RP2354A** 是一款以教育研究為導向的四軸飛行器飛控板，核心採用 Raspberry Pi **RP2354A** 微控制器。設計目標是讓學生與研究人員能夠深入接觸現代飛控軟體堆疊的完整面貌——從底層硬體驅動、訊號處理、感測器融合，到閉迴路控制——在 **Arduino 2.3.x** 生態系中即可開發與燒錄。

| 功能項目 | 規格說明 |
|---|---|
| 微控制器 | RP2354A — Cortex-M33 雙核心，150 MHz |
| 慣性量測單元（IMU） | Bosch BMI088 — 加速度計與陀螺儀各自獨立晶片，SPI 介面，DRDY 最高 2 kHz |
| 氣壓計 | Bosch BMP580 — I²C 介面，50 Hz，24 位元氣壓 + 溫度 |
| 遙控接收器 | ExpressLRS / CRSF — UART 介面，420 kBaud，16 頻道 |
| 有刷馬達輸出 | 4 路 MOSFET PWM（GPIO 29 / 11 / 18 / 25，32 kHz，10 位元） |
| 無刷 ESC 輸出 | 4 路 DShot600 PIO（GPIO 1 / 4 / 10 / 14） |
| 飛行黑盒儲存 | W25Q128 SPI Flash，16 MiB *(選配——現行板卡版本未焊接)* |
| 電池電壓偵測 | 12 位元 ADC — 分壓電路於 GPIO 27 *(電流偵測未實作)* |
| 開發環境 | Arduino IDE 2.3.8 + earlephilhower/arduino-pico ≥ 3.x |

## 2. 雙核心執行架構

RP2354A 整合了**兩個相互獨立的 Cortex-M33 處理器**，共享系統記憶體，但各自執行完全不同的指令流。韌體將工作分配為：**Core 0** 負責硬即時飛行控制，**Core 1** 負責 I/O 密集型任務，使得任何周邊活動——UART 解析、遙測格式化、FFT 計算——都不會干擾到關鍵的姿態控制迴路。

| Core 0（150 MHz）— 硬即時域 | Core 1（150 MHz）— I/O 與分析域 |
|---|---|
| BMI088 DRDY 中斷服務程式（ISR）@ 2 kHz | ELRS / CRSF 封包解碼（非同步） |
| → RPM 陷波濾波器組 | CMSIS-DSP FFT @ ~8 Hz |
| → 雙二階低通濾波器（三軸） | → 更新動態陷波中心頻率 |
| → Madgwick AHRS 四元數更新 | |
| **1 kHz 控制迴路：** | 透過 `volatile` 原子拷貝與 Core 0 共享資料 |
| → 高度估算 | |
| → 飛行模式邏輯 | |
| → 雙迴路 PID（角速率 + 角度） | |
| → Quad-X 馬達混控 | |
| → DShot600 PIO / 有刷 PWM 輸出 | |
| → 黑盒寫入 *(選配)* | |
| → USB 遙測 @ 100 Hz | |

### 2.1 Core 0 — 硬即時域

Core 0 負責所有時間關鍵操作。BMI088 陀螺儀在**精確 2 000 Hz** 觸發 Data-Ready（DRDY）中斷；Core 0 服務此 ISR、執行完整的陀螺儀濾波鏈，並更新 Madgwick 四元數。隨後的 **1 kHz 輪詢迴路**讀取最新 AHRS 輸出，計算 PID 修正量，混控後送往有刷 PWM 周邊或 PIO DShot 狀態機。

整個控制路徑——從感測器到致動器——在 **≈ 50 µs** 內完成，即便啟用所有選配功能，Core 0 仍有超過 85 % 的閒置時間。

### 2.2 Core 1 — I/O 與分析域

Core 1 執行無限迴圈，持續排空 CRSF UART 接收 FIFO，解碼 ExpressLRS 封包並更新共享的 RC 頻道表。選配情況下，它還會將陀螺儀樣本送入 256 點 CMSIS-DSP 實數 FFT，偵測主要振動頻率，並透過 `volatile` 原子變數將更新後的陷波濾波器中心頻率回傳給 Core 0。

由於 Core 1 **從不接觸馬達輸出或 PID 狀態**，Core 1 上的任何延遲對控制迴路而言都是完全透明的。

> **核心隔離的關鍵效益：** UART 在 420 kBaud 峰值速率下每隔 24 µs 就會產生一次中斷。將 CRSF 解析限制在 Core 1，這些中斷對 Core 0 上的 2 kHz 陀螺儀 ISR 來說完全不可見——消除了單核心飛控必須以中斷優先權機制處理的主要抖動來源。

## 3. 姿態與航向參考系統（AHRS）

飛控板實作了 **Sebastian Madgwick 梯度下降姿態濾波器**（2010），將 BMI088 陀螺儀與加速度計資料融合為單位四元數，即時表示機體在三維空間中的完整姿態。

### 3.1 為何採用四元數表示法？

尤拉角（roll、pitch、yaw）在兩個旋轉軸對齊時會發生**萬向鎖死（gimbal lock）**問題。單位四元數 **q = [q₀, q₁, q₂, q₃]** 可在無奇異點的情況下表示任意姿態，只需四個浮點數，且姿態合成僅需一次乘法——遠比串接 3×3 旋轉矩陣高效。

### 3.2 Madgwick 濾波演算法

每次感測器採樣（2 000 Hz）時，濾波器依序執行以下六個步驟：

| 步驟 | 說明 | 公式 |
|:---:|---|---|
| 1 | **角速率積分** — 以陀螺儀角速率將四元數向前推進 | `q̇ = 0.5 · q ⊗ ωgyro` |
| 2 | **計算目標函數** — 量測重力方向與預測方向的偏差 | `f = q ⊗ g_ref ⊗ q* − g_meas` |
| 3 | **梯度下降** — 計算雅可比矩陣並正規化梯度 | `∇f = Jᵀ · f` |
| 4 | **修正** — 以融合增益 β 縮放誤差梯度 | `q̇_err = β · ∇f / ‖∇f‖` |
| 5 | **合併** — 結合角速率積分與修正量 | `q̇_total = q̇ − q̇_err` |
| 6 | **積分與正規化** — 更新四元數並保持單位長度 | `q += q̇_total / fs;  q /= ‖q‖` |

最終從四元數提取尤拉角（roll、pitch、yaw）供 PID 控制器使用。融合增益 **β** 在陀螺儀響應速度與加速度計雜訊修正之間取得平衡——值越大修正越積極，值越小越依賴陀螺儀積分。

### 3.3 FPU 硬體加速

每次 Madgwick 更新需約 **30 次單精度浮點運算**。Cortex-M33 FPU 使其在 2 kHz 下可行：

| 運算 | Cortex-M33 FPU | 軟體模擬（M0+） |
|---|---|---|
| F32 乘加（VMLA） | 1 個時脈週期（流水線） | ~5 個時脈週期 |
| F32 加法 / 減法 | 1 個時脈週期（流水線） | ~5 個時脈週期 |
| `VSQRT.F32`（硬體） | 14 個時脈週期 | ~400 個時脈週期 |
| 完整 Madgwick 更新 | ≈ 0.5 µs（150 MHz） | ≈ 8 µs（估算） |
| 2 kHz 下的 CPU 佔用 | **< 0.1 %** | ~1.6 % |

若無 FPU，M0+ 核心上的軟體 `sqrt()` 需耗費 ≈ 400 個時脈週期——僅一個運算就有 **28 倍的效能損失**。

## 4. 雙迴路 PID 飛行控制

韌體實作了所有現代 Betaflight 級飛控所採用的**雙迴路（串聯）PID 架構**。外迴路穩定機體姿態（角度），內迴路調節角速率。兩個迴路以不同頻率運行並使用不同感測器訊號，賦予飛行器既穩定的自動水平又快速的擾動抑制能力。

### 4.1 整體訊號流

| 層級 | 模式 | 輸入 | 運算 | 輸出 |
|:---:|---|---|---|---|
| **遙控搖桿** | 全部 | 操控者 | — | 正規化值 ±1.0 |
| ↓ | | | | |
| **外迴路**（角度 PID） | 角度模式 | 搖桿值 × 最大角度（45°） | `error = targetAngle − currentAngle` | 角速率設定值（°/s） |
| **外迴路** | 特技模式 | 搖桿值 × 最大角速率（600 °/s） | 直接對應，跳過角度 PID | 角速率設定值（°/s） |
| ↓ | | | | |
| **內迴路**（角速率 PID） | 全部 | 角速率設定值 vs. BMI088 陀螺實測值 | `PID = Kp·e + Ki·∫e dt + Kd·(de/dt)` | 三軸修正量 |
| ↓ | | | | |
| **Quad-X 馬達混控** | 全部 | 三軸修正量 + 油門 T | M1=T+roll+pitch−yaw … M4=T+roll−pitch+yaw | 馬達指令 0–1 |
| ↓ | | | | |
| **馬達輸出** | 編譯期選擇 | 馬達指令 | — | 有刷 PWM 或 DShot600 PIO |

### 4.2 內迴路——角速率控制（1 kHz）

| 項目 | 說明 |
|---|---|
| **感測器** | BMI088 陀螺儀，經雙二階 LPF + RPM 陷波濾波器組處理 |
| **輸入** | 期望角速率（rad/s），來自外迴路或特技模式下的搖桿直接對應 |
| **輸出** | 正規化馬達修正量，混控後限制於 0–1 |

**微分項**對量測值（而非誤差）進行微分，以避免設定值階躍時產生「微分衝擊（derivative kick）」。**積分抗積分飽和鉗制**防止馬達飽和時積分器失控。

### 4.3 外迴路——角度穩定（角度模式）

Madgwick 濾波器輸出角度制的 roll 與 pitch。外迴路的 **P 控制器**計算期望傾斜角（與搖桿偏轉量成正比，限制在 ±45°）與量測傾斜角之間的誤差，並將其縮放為角速率指令送往內迴路。

這種串聯結構使**內迴路無論在哪種模式下都在角速率空間中工作**，讓特技模式與角度模式之間的 PID 調整保持一致。

### 4.4 高度保持——氣壓感測器融合

一個獨立的垂直軸 PID 基於**互補濾波器**估算的高度（公尺，相對於起飛點）運行，融合 BMP580 氣壓高度與 Z 軸加速度計積分：

| 訊號來源 | 特性 | 作用 |
|---|---|---|
| 氣壓計（BMP580） | 低頻、絕對高度 | 長期漂移修正 |
| 加速度計（BMI088 Z 軸） | 高頻、即時速度 | 動態響應 |

透過 CH7 開關啟用高度保持後，此 PID 的輸出將取代原始油門指令。

### 4.5 Ziegler-Nichols 自動調參

韌體內建適用於台架測試的**繼電器回授自動調參器**。它對零角速率施加開關振盪，量測極限週期 *Pᵤ* 與振盪幅值 *A*，計算極限增益 *Kᵤ = 4d / (π·A)*，再套用 Ziegler-Nichols PID 公式：

```
Kp = 0.6 · Ku
Ki = 2 · Kp / Pu
Kd = Kp · Pu / 8
```

所得結果作為實際機體手動細調的初始增益。

## 5. 可切換馬達輸出——空心杯有刷 vs. BLDC 無刷

RB-RP2354A 在同一套硬體上支援兩種根本不同的四軸飛行器類型，透過 `config.h` 中的單一 `#define` 在**編譯期**選擇馬達驅動方案。

| 屬性 | 空心杯有刷（`MOTOR_TYPE_BRUSHED`） | BLDC 無刷 DShot600（`MOTOR_TYPE_BLDC`） |
|---|---|---|
| GPIO 腳位 | 29、11、18、25 | 1、4、10、14 |
| 驅動電路 | 每顆馬達配一顆 N-MOSFET | 外接 BLHeli / AM32 ESC |
| 通訊協定 | PWM 32 kHz，10 位元 | DShot600 數位協定，600 kbit/s |
| 油門解析度 | 1 024 階 | 2 000 階（11 位元） |
| ESC 校準 | 不需要 | ESC 行程校準或 DShot（自動） |
| RPM 回傳 | 無 | 雙向 DShot eRPM 或 BLHeli 序列遙測 |
| 典型機型 | 65 mm TinyWhoop / 微型有刷機 | 2″–5″ 競速機或特技機 |

### 5.1 有刷馬達路徑——32 kHz PWM

空心杯馬達由 RP2354A GPIO 腳位直接驅動單顆邏輯電平 N-MOSFET。PWM 頻率設為 **32 kHz**——高於人耳可聽頻率，亦遠高於 BMI088 加速度計通頻帶——因此開關雜訊對 IMU 不可見。`analogWriteResolution(10)` 提供 1 024 階油門解析度，對微型機的控制完全足夠。

### 5.2 無刷馬達路徑——DShot600 透過 PIO

無刷馬達需要外接 ESC 並以數位脈衝協定驅動。DShot600 將每道油門指令編碼為 **16 位元封包**（11 位元油門值、1 位元遙測請求、4 位元 CRC），以 600 kbit/s 傳輸。韌體透過 RP2354A **PIO 硬體**（詳見[第 8 節](#8-可程式-iopio精準馬達訊號輸出)）實作 DShot600，可同時輸出四路 ESC 訊號，次奈秒級抖動，**CPU 零介入**。

## 6. DSP 與 FPU——加速複雜數學運算

RP2354A 內的 Cortex-M33 核心包含 **ARMv8-M 浮點運算單元（FPU）** 與選配的 **DSP 指令集擴充**。兩者對於在微控制器上以千赫茲速率執行完整飛控堆疊不可或缺。

### 6.1 FPU——硬體單精度浮點

所有感測器數值與中間計算結果均以 IEEE 754 單精度浮點數儲存。主要指令延遲如下：

| 運算 | Cortex-M33 FPU | 軟體模擬（M0+） | 加速倍數 |
|---|---|---|---|
| `VMLA.F32` 乘加 | 1 個時脈週期 | ~5 個時脈週期 | 5× |
| `VADD.F32` / `VSUB.F32` | 1 個時脈週期 | ~5 個時脈週期 | 5× |
| `VSQRT.F32` | 14 個時脈週期 | ~400 個時脈週期 | **28×** |
| `VDIV.F32` | 14 個時脈週期 | ~200 個時脈週期 | **14×** |

執行約 60 次浮點運算的 Madgwick AHRS 更新在 150 MHz 下耗時**不到 1 µs**——在 1 ms 的控制迴路中幾乎可忽略不計。

### 6.2 DSP 擴充——SIMD 整數運算

M33 DSP 擴充新增了 SIMD 指令，可將 32 位元暫存器視為兩個 16 位元或四個 8 位元通道同時運算。本韌體的 DSP 擴充加速三個領域：

- **CRSF 頻道解包**：橫跨位元組邊界打包的 11 位元值，以 `UXTB16` / `UBFX` 在無分支的緊湊迴圈中提取。
- **雙二階濾波器鏈**：`SMLAD` / `SMLALD` 指令每個時脈週期可執行兩次乘加，用於 CMSIS-DSP 程式庫路徑下的 Q15 濾波實作。
- **FFT 蝴蝶運算**：`arm_rfft_fast_f32` 利用 ARMv8-M SIMD 本質指令（intrinsics）執行 radix-4 蝴蝶運算的順序重排。

### 6.3 CMSIS-DSP 程式庫整合

選配的 FFT 振動分析模組（`fft_cmsis.cpp`）使用 ARM CMSIS-DSP 的 **`arm_rfft_fast_f32`** 函式，其實作針對 Cortex-M33 使用 ARMv8-M DSP 本質指令最佳化的分裂基數實數 FFT：

| 實作方式 | 時間複雜度 | N=256 計算時間 | 8 Hz 下 CPU 佔用 |
|---|---|---|---|
| 暴力 DFT | O(N²) | ≈ 34 000 µs | ~27 %（無法使用） |
| CMSIS-DSP `arm_rfft_fast_f32` | O(N log₂N) | ≈ 240 µs | **< 0.2 %** |
| **加速倍數** | — | **140×** | — |

這使得在 Core 1 上進行即時振動頻率追蹤成為可能，且完全不影響 Core 0 的飛行控制迴路。

## 7. 硬體 FIFO——確保控制迴路不中斷

RP2354A 上每個時間敏感周邊——UART、SPI、PIO——都在周邊邏輯與 CPU 之間內建了**硬體 FIFO**。FIFO 使飛行控制迴路得以在不被每個抵達或送出的位元組中斷的情況下運行。

### 7.1 UART FIFO——吸收 CRSF 突發資料

ELRS 接收器每 ~4 ms 以 420 kBaud 傳送一個 **26 位元組的 CRSF 封包**。在 420 000 baud 下，每個位元組每隔 23.8 µs 到達一次。RP2354A UART0 內建 **32 位元組 RX FIFO**，使整個 CRSF 封包得以在硬體緩衝區中等候，完全不產生中斷。

資料流路徑如下：

1. **ELRS 發射模組** 傳送 26 位元組封包（≈ 619 µs）
2. → **UART0 RX 硬體 FIFO（32 B）** 自動填充，無需中斷
3. → **Core 1** 空閒時主動排空 FIFO
4. → **CRSF 位元組解析器** 解碼 16 個頻道
5. → 更新 `volatile` RC 頻道表（由 Core 0 原子讀取）

Core 1 在 FFT 更新之間機動地排空 FIFO——不需要緊湊的 ISR，也完全不涉及 Core 0。

### 7.2 SPI FIFO——非阻塞感測器讀取

在 10 MHz SPI 下，一次 BMI088 陀螺儀 6 位元組連續讀取耗時 **≈ 6 µs**。RP2354A SPI TX 與 RX FIFO 各深 **8 項 × 16 位元**。ISR 預先將整個 7 位元組事務（1 位元組命令 + 6 位元組資料）載入 TX FIFO，SPI 硬體自動將其送出並填充 RX FIFO，ISR 最後才讀取 RX FIFO——SPI 傳輸期間 **CPU 可自由執行其他指令**。

### 7.3 PIO TX FIFO——DShot 零 CPU 介入

每個 PIO 狀態機都有自己的 **4 項 TX FIFO**。1 kHz 迴路計算完馬達指令後，將四個 32 位元 DShot 封包寫入各自的 PIO FIFO——**四條 32 位元 store 指令，共 ≈ 8 個 CPU 時脈週期**。PIO 狀態機獨立排空 FIFO，以精確 600 kbit/s 逐位元輸出 DShot 封包，無需 CPU 再次介入。

若 CPU 因某個 ISR 短暫延遲，FIFO 會吸收這段抖動——馬達訊號仍保持位元完美。

> **FIFO 深度對飛行控制的重要性：** 1 kHz 控制迴路每兩次執行之間僅有 1 000 µs。若某個周邊需要每 24 µs 輪詢一次（420 kBaud 下的單一 CRSF 位元組），主迴路每毫秒就會被中斷 42 次——使任何確定性時序都無從實現。硬體 FIFO 將工作批次化為粗粒度的工作塊，自然契合迴路節奏，這也是即時嵌入式系統設計中善用 FIFO 的根本原因。

## 8. 可程式 I/O（PIO）——精準馬達訊號輸出

RP2354A 包含**兩個 PIO 區塊**，各含**四個獨立狀態機**。每個狀態機是一個微型可程式處理器，擁有自己的指令記憶體、移位暫存器與時脈分頻器。PIO 狀態機與兩個 Cortex-M33 核心平行運行，並可直接、**時脈精準地存取 GPIO 腳位**——非常適合產生軟體中斷抖動無法容忍的時序關鍵波形。

### 8.1 PIO 上的 DShot600——架構說明

DShot600 要求每個位元週期精確為 **1 667 ns**（= 1/600 kHz），容差 ±5 %（±83 ns）：

| 訊號位元 | 高電位時間（T_H） | 低電位時間（T_L） |
|:---:|---|---|
| 邏輯 1 | 1 250 ns | 417 ns |
| 邏輯 0 | 625 ns | 1 042 ns |

執行搶佔式排程器的軟體位元鎖（bit-bang）迴路很容易超出此容差。PIO 狀態機執行專用程式，以**時脈精準的時序**產生每個位元，完全獨立於 Cortex-M33 核心正在執行的任何工作。

**執行流程**如下：

1. **Core 0 控制迴路（1 kHz）**：計算馬達指令 → 編碼 DShot 封包 → 推入 PIO FIFO（4 條 `store` 指令，≈ 8 個時脈週期，Core 0 隨即釋放）
2. **PIO0 SM0**（ESC1 / GPIO 1）：從 FIFO 取得封包，對 16 個位元依序輸出精確波形
3. **PIO0 SM1**（ESC2 / GPIO 4）：同上，與 SM0 精確同步
4. **PIO0 SM2**（ESC3 / GPIO 10）：同上
5. **PIO0 SM3**（ESC4 / GPIO 14）：同上

四路訊號同時發出——啟動時間差在一個 PIO 時脈週期（< 7 ns）以內。

### 8.2 時序精準度

PIO 時脈分頻器設定使每個 PIO 時脈刻度 = **104.2 ns**（150 MHz ÷ 15.625）。DShot 位元週期使用 **16 個 PIO 刻度 = 1 667 ns**，與 DShot600 規格完全吻合。四個狀態機共享同一份載入至 PIO0 指令記憶體的 PIO 程式，以完美同步方式運行。

### 8.3 對雙核心 CPU 負載的影響

| 馬達輸出方式 | 每次 1 kHz 更新的 CPU 時間 | 備註 |
|---|---|---|
| 軟體位元鎖（bit-bang） | ≈ 106.8 µs | 4 顆馬達 × 26.7 µs，期間須關閉中斷 |
| PIO 狀態機 | ≈ 0.027 µs | 4 × FIFO `store` 指令 |
| **節省倍數** | **≈ 3 960 倍** | 釋放資源給濾波器、黑盒、MSP 遙測 |

若不使用 PIO，軟體位元鎖 DShot600 驅動僅馬達輸出就佔用 Core 0 超過 **10 %**。使用 PIO 後這一開銷降至實質上的**零**。

## 9. 陀螺儀濾波流水線

陀螺儀原始資料包含兩類主要雜訊：來自感測器本身的寬頻電子雜訊，以及馬達與螺旋槳諧振產生的離散振動峰值。濾波鏈在保留靈敏姿態控制所需頻寬的同時，去除上述兩類雜訊。

| 處理階段 | 條件 | 說明 |
|---|---|---|
| **① 偏差量扣除** | 必要 | 開機靜置 2 秒校準，減去陀螺儀零偏 |
| **② RPM 陷波濾波器組** | 啟用 `USE_RPM_FILTER` | 8 個陷波器（4 顆馬達 × 基頻 + 二次諧波），依 ESC 回傳 eRPM 每 ~4 ms 更新中心頻率 |
| **③ 動態陷波濾波器** | 啟用 `USE_FFT_ANALYSIS` | 1 個陷波器，中心頻率由 Core 1 CMSIS-DSP FFT 每 ~50 ms 更新 |
| **④ 雙二階低通濾波器（LPF）** | 必要（各軸獨立） | 巴特沃斯二階，截止頻率 150 Hz，採樣率 2 kHz |
| ↓ **輸出至** | — | Madgwick AHRS + PID 內迴路 |

每個**雙二階濾波器**採用 **Direct Form II 轉置形式**實作——此為數值穩定的形式，在截止頻率與採樣率比值僅 0.075（150 Hz / 2 000 Hz）的低比值情況下，能以單精度浮點數將係數靈敏度誤差降到最低。

## 10. 遙測、飛行記錄與安全系統

### 10.1 USB 遙測與 MSP 協定

韌體透過 USB 序列埠以 **100 Hz** 傳送包含 **22 欄位的 CSV 封包**：

```
roll×10, pitch×10, yaw×10,
m1, m2, m3, m4,       （0–1000）
kP, kI, kD,
alt_cm, armed, mode,
volt_mV, curr_cA, bat_pct,
r1rpm, r2rpm, r3rpm, r4rpm,
loop_us, fs_level
```

同一序列埠也回應 **MSP v1**（MultiWii Serial Protocol）封包，使官方 **Betaflight Configurator** 得以連線顯示即時姿態、馬達測試及 PID 調整。`$M` 前導位元組可自動識別 MSP 封包，因此兩種協定無需配置即可共存。

**Processing 4 圖形介面**（`gui/mini_bf_config.pde`）提供四頁標籤式操作介面：

| 頁籤 | 內容 |
|---|---|
| 飛行 | 3D 姿態立方體、馬達輸出條 + ESC RPM、電池面板、FFT 頻譜 |
| PID 調整 | Roll/Pitch 與 Yaw 滑桿及即時 FC 數據回饋；調整技巧說明 |
| 遙控校準 | 各頻道 expo 曲線與死區調整；開始/結束掃描按鈕 |
| 自我測試 | 即時硬體驗證，附通過/失敗標示 |

### 10.2 飛行黑盒紀錄器 *(選配——需外接 W25Q128)*

> **硬體說明：** 現行 RB-RP2354A 板卡版本**未焊接** W25Q128 SPI Flash。黑盒子系統在韌體中已完整實作，可透過將 W25Q128JV 焊接至 SPI 焊墊（CS = GPIO 28）並在 `config.h` 中加入 `#define USE_BLACKBOX` 來啟用。以下說明適用於 Flash 已安裝的情況。

安裝 Flash 後，飛行器解鎖期間將以 **1 kHz** 向 W25Q128 SPI Flash 寫入緊湊的 **32 位元組二進位記錄幀**，每幀記錄：

```c
uint32_t timestamp_us;
int16_t  roll_x10, pitch_x10, yaw_x10;    // 角度 × 10
int16_t  gx_x100, gy_x100, gz_x100;       // rad/s × 100
uint16_t m1, m2, m3, m4;                  // 馬達輸出 0–1000
uint16_t altitude_cm;
uint16_t voltage_mV;
uint8_t  armed_mode;
uint8_t  flags;
uint16_t loop_time_us;
// 合計：32 位元組
```

**頁寫入緩衝**（累積 8 幀後才執行一次 256 位元組 Flash 頁寫入）使寫入延遲保持在 700 µs 以下。以 1 kHz 速率，W25Q128 每個飛行任務可儲存 ≈ **524 秒**的飛行資料。Python 解碼腳本（`blackbox/decode.py`）可將原始轉儲轉換為 CSV，供 Excel 或 pandas 分析。

```bash
# 透過 USB 轉儲 Flash
python3 blackbox/dump_flash.py /dev/ttyACM0 flash.bin

# 解碼所有任務為 CSV
python3 blackbox/decode.py flash.bin
```

### 10.3 四級故障保護

韌體持續監控 CRSF 連線時間與電池電壓：

| 等級 | 觸發條件 | 回應動作 |
|---|---|---|
| `NONE` | 一切正常 | 正常飛行 |
| `WARN` | 連線中斷 > 100 ms 或電池電量低 | 僅警報 |
| `LAND` | 連線中斷 > 500 ms 或電池電量嚴重不足 | 油門在 8 秒內線性降至零 |
| `KILL` | 連線中斷 > 2 s 或降落完成 | 立即停止馬達 + 解鎖 |

### 10.4 開機硬體自我測試

飛行迴路啟動前，硬體自我測試依序驗證以下八項：

| # | 項目 | 預期結果 |
|:---:|---|---|
| 1 | BMI088 加速度計晶片 ID（SPI） | `0x1E` |
| 2 | BMI088 陀螺儀晶片 ID（SPI） | `0x0F` |
| 3 | BMP580 氣壓計晶片 ID（I²C） | `0x50` |
| 4 | W25Q128 Flash JEDEC ID（SPI） | `0xEF4018` — *未定義 `USE_BLACKBOX` 時跳過* |
| 5 | 電池電壓範圍 | 3.0 V – 25.2 V |
| 6 | CRSF 連線訊號 | 2 秒內收到 |
| 7 | 陀螺儀靜置雜訊 | RMS < 0.05 rad/s |
| 8 | 加速度計重力量值 | 0.90 – 1.10 g（板卡水平） |

測試結果輸出至序列埠，並由 GUI 自我測試頁籤解析顯示。

## 11. PID 增益調整指南

### 11.1 增益單位說明

在調整數值之前，必須先理解各增益作用在哪個單位上——否則數字本身沒有意義。

| 迴路 | 量測 / 設定值單位 | 輸出單位 |
|---|---|---|
| **內迴路（角速率 PID）** | rad/s（BMI088 陀螺儀經 `GYRO_SCALE` 縮放後） | 無因次馬達修正量，±`PID_OUTPUT_LIMIT`（±500） |
| **外迴路（角度 P）** | 度（Madgwick 輸出的尤拉角） | deg/s 角速率指令，送往內迴路 |

舉例：`Kp_inner = 2.0` 代表 **1 rad/s（≈ 57°/s）的角速率誤差**產生 **2.0 的馬達修正量**（相對於 ±500 的輸出限幅，屬於很溫和的響應，適合作為起始值）。

`Kp_outer = 6.0` 代表 **10° 的傾斜誤差**產生 **60 deg/s 的角速率指令**送往內迴路。

---

### 11.2 預設增益建議

`config.h` 內置兩組預設值，使用時取消對應區塊的注釋，並注釋另一組即可。

#### Preset A：1S–2S 有刷空心杯  30–60 g（65 mm TinyWhoop 級別）

空心杯機體**轉動慣量極小**，馬達響應較慢（約 20 ms），對陀螺儀雜訊中的 Kd 項較敏感，應從較保守的值開始。

| 參數 | 值 | 說明 |
|---|---|---|
| `PID_ROLL_KP` | **2.0** | 保守起點，確認陀螺校準後可提升 |
| `PID_ROLL_KI` | **0.04** | 修正慢速漂移 |
| `PID_ROLL_KD` | **0.015** | 低值避免空心杯振動被放大 |
| `PID_PITCH_KP/KI/KD` | **同 Roll** | 通常與 Roll 相同 |
| `PID_YAW_KP` | **2.5** | |
| `PID_YAW_KI` | **0.04** | |
| `PID_YAW_KD` | **0.00** | Yaw D 在有刷機上幾乎必為零——只會引入噪音 |
| `OUTER_LOOP_KP` | **6.0** | 10° 誤差 → 60 deg/s 速率指令 |
| `MAX_ANGLE_DEG` | **30°** | 較窄的傾斜限制，適合室內微型飛行 |
| `MAX_RATE_DPS` | **400 °/s** | 特技模式最大角速率 |

#### Preset B：2S–3S BLDC 無刷  80–180 g（2″–3″ 牙籤機 / 微型競速機）

無刷機有**更高的轉動慣量**，馬達響應更快（DShot 約 5 ms），結構性振動較規律，需要更高的增益才能達到有效的擾動抑制。

| 參數 | 值 | 說明 |
|---|---|---|
| `PID_ROLL_KP` | **4.0** | BLDC 需要更高的 P 值來抵抗螺旋槳尾流 |
| `PID_ROLL_KI` | **0.06** | |
| `PID_ROLL_KD` | **0.035** | 較高的 D 值利用 DShot 快速響應抑制振盪 |
| `PID_PITCH_KP/KI/KD` | **同 Roll** | |
| `PID_YAW_KP` | **5.0** | Yaw 的控制權威主要來自 P 值 |
| `PID_YAW_KI` | **0.06** | |
| `PID_YAW_KD` | **0.00** | Yaw D 仍為零——即使對 BLDC 也會引起抖動 |
| `OUTER_LOOP_KP` | **10.0** | 較快速的角度響應 |
| `MAX_ANGLE_DEG` | **45°** | 標準傾斜限制 |
| `MAX_RATE_DPS` | **600 °/s** | 標準特技模式速率 |

#### 兩組預設值對照

| 參數 | Preset A（有刷 30–60 g） | Preset B（BLDC 80–180 g） |
|---|---|---|
| `PID_ROLL_KP` | 2.0 | 4.0 |
| `PID_ROLL_KI` | 0.04 | 0.06 |
| `PID_ROLL_KD` | 0.015 | 0.035 |
| `PID_YAW_KP` | 2.5 | 5.0 |
| `PID_YAW_KD` | **0.00** | **0.00** |
| `OUTER_LOOP_KP` | 6.0 | 10.0 |
| `MAX_ANGLE_DEG` | 30° | 45° |

編輯 `config.h`，取消目標預設塊的注釋，並注釋另一塊：

```c

```

```c

```

### 11.4 重量比例估算公式

適用於 40–180 g 之間任何機型的線性插值估算：

| 增益 | 估算公式 |
|---|---|
| `Kp_inner` | `2.0 + (機重g − 40) × 0.033` |
| `Kd_inner`（有刷） | `Kp_inner × 0.008` |
| `Kd_inner`（BLDC） | `Kp_inner × 0.009` |
| `Ki_inner` | `Kp_inner × 0.015` |
| `Kp_outer` | `6.0 + (機重g − 40) × 0.025` |

**驗算範例**（100 g BLDC 機）：

```
Kp_inner  = 2.0 + (100 - 40) x 0.033  = 2.0 + 1.98  = 4.0
Kd_inner  = 4.0 x 0.009               = 0.036
Ki_inner  = 4.0 x 0.015               = 0.060
Kp_outer  = 6.0 + (100 - 40) x 0.025 = 6.0 + 1.50  = 7.5
```

| 增益 | 計算結果 | 與 Preset B 對照 |
|---|---|---|
| `Kp_inner` | 4.0 | Preset B = 4.0 ✓ |
| `Kd_inner` | 0.036 | Preset B = 0.035 ≈ ✓ |
| `Ki_inner` | 0.060 | Preset B = 0.06 ✓ |
| `Kp_outer` | 7.5 | Preset B = 10.0（競速機偏好更高響應性） |

### 11.5 分步調整程序

> **前提：** 所有 PID 調整必須在陀螺儀校準通過之後進行。開機靜置 2 秒，確認序列監視器顯示 `[FC] Calibration done` 且 Self-Test 頁籤的「陀螺儀靜置雜訊」為 **PASS**。

#### 第一階段：內迴路（特技模式，先將 Ki = 0、Kd = 0）

1. **只調 Kp。** 懸停觀察：

   | 現象 | 診斷 | 處理 |
   |---|---|---|
   | 機體緩慢漂移或無法保持水平 | Kp 太低 | 增加 0.5 步 |
   | 高頻快速震盪（嗡嗡聲） | Kp 太高 | 降低 0.3 步 |
   | 基本保持水平，輕微低頻擺動 | Kp 合適，繼續下一步 | — |

2. **加入 Kd**（初始值 = Kp × 0.008 有刷 / Kp × 0.009 BLDC）。Kd 抑制 Kp 引起的震盪：

   | 現象 | 診斷 | 處理 |
   |---|---|---|
   | 落地或螺旋槳尾流時馬達嗡嗡震動 | Kd 太高 | 減半 |
   | Kp 步驟的震盪仍存在 | Kd 太低或 Kp 仍過高 | 先降低 Kp |

3. **加入 Ki**（初始值 = Kp × 0.015）。Ki 消除上一步之後的緩慢漂移：

   | 現象 | 診斷 | 處理 |
   |---|---|---|
   | 緩慢積累的搖擺越來越大 | Ki 太高（積分飽和） | 降低 0.01 步 |
   | 懸停時仍向一側漂移 | Ki 太低 | 增加 0.01 步 |

#### 第二階段：外迴路（角度模式，內迴路穩定後）

切換至角度模式，調整 `OUTER_LOOP_KP`（在 `config.h` 中）：

| 現象 | 診斷 | 處理 |
|---|---|---|
| 自動水平，但會超過水平並輕微反向搖擺 | `Kp_outer` 太高 | 降低 1.0 步 |
| 回到水平需要 > 1 秒，感覺遲鈍 | `Kp_outer` 太低 | 增加 1.0 步 |
| 0.3–0.5 秒內平滑回到水平，無超調 | 正確 | — |

#### 第三階段：啟用濾波器（如有振動問題）

| 現象 | 解決方案 |
|---|---|
| 特定油門點出現高頻馬達震動 | 啟用 `USE_RPM_FILTER`（需 ESC eRPM 遙測） |
| 振動頻率隨油門變化 | 啟用 `USE_FFT_ANALYSIS`，動態陷波自動追蹤 |
| 普遍性抖動或不穩定 | 將 `GYRO_LPF_CUTOFF_HZ` 從 150 Hz 降至 100 Hz |

## 附錄 A：CMSIS-DSP FFT @ ~8 Hz 動態陷波更新機制說明

> 本附錄對應第 2 節雙核心架構表格中 **\*** 標記的項目：  
> *「CMSIS-DSP FFT @ ~8 Hz → 更新動態陷波中心頻率」*

---

### A.1 問題背景——為何需要動態陷波？

馬達高速旋轉時，螺旋槳的不平衡與機架諧振會產生振動，並透過機架傳遞至 IMU，在陀螺儀訊號上疊加週期性雜訊峰值。這個峰值的**頻率隨油門變化**——油門越高，馬達轉速越快，振動頻率越高。

固定頻率的陷波濾波器只能消除單一頻率，一旦 RPM 改變就會失效。**動態陷波濾波器**持續追蹤並跟隨這個峰值頻率，才能在整個油門範圍內有效抑制振動雜訊。

---

### A.2 FFT 為何只需 ~8 Hz 更新？

答案來自 FFT 視窗本身的大小與陀螺儀採樣率的關係。

**視窗填充時間**

韌體在 Core 1 維護一個 **256 點**的循環緩衝區，由 Core 1 每次從 `bmi088GetLatest()` 取得最新陀螺儀樣本後填入。陀螺儀的採樣率為 2 000 Hz，因此填滿 256 個樣本所需的時間為：

> **256 個樣本 ÷ 2 000 Hz = 0.128 秒 ≈ 128 ms**

緩衝區**每填滿一次就觸發一次 FFT 計算**，每秒觸發次數為：

> **1 ÷ 0.128 s ≈ 7.8 Hz ≈ ~8 Hz**

這個 ~8 Hz **不是定時器設定**，而是視窗大小與採樣率自然決定的結果。

**頻率解析度**

256 點 FFT 在 2 000 Hz 採樣率下，每個頻率箱（bin）的寬度為：

> **頻率解析度 = 採樣率 ÷ FFT 點數 = 2 000 Hz ÷ 256 = 7.8 Hz / bin**

這意味著可以分辨相差 7.8 Hz 以上的振動峰值，對於追蹤馬達 RPM 引起的振動（通常在 50–800 Hz 之間）已完全足夠。

---

### A.3 執行流程

下表說明從陀螺儀採樣到陷波濾波器更新的完整路徑：

| 步驟 | 執行核心 | 頻率 | 說明 |
|:---:|---|---|---|
| ① BMI088 DRDY ISR 觸發 | Core 0 | 2 000 Hz | 陀螺儀每 0.5 ms 產生一次 Data-Ready 中斷 |
| ② `bmi088GetLatest()` | Core 1 | 2 000 Hz | Core 1 讀取最新樣本，送入 256 點循環緩衝區 |
| ③ 緩衝區填滿 | Core 1 | ~7.8 Hz | 每 128 ms 填滿一次，觸發 FFT |
| ④ `arm_rfft_fast_f32()` | Core 1 | ~7.8 Hz | CMSIS-DSP 計算 256 點實數 FFT，耗時 ≈ 240 µs |
| ⑤ 掃描頻率箱，找峰值 | Core 1 | ~7.8 Hz | 忽略 < 20 Hz 的低頻箱（直流與低頻干擾），找最大幅值箱 |
| ⑥ 計算峰值頻率 | Core 1 | ~7.8 Hz | `peakHz = maxBin × 7.8 Hz` |
| ⑦ 更新陷波中心頻率 | Core 1 → Core 0 | ~7.8 Hz | 透過 `volatile` 原子變數傳遞給 Core 0 的陷波濾波器 |
| ⑧ 重新初始化陷波器 | Core 0 | 下次濾波迴路 | `biquadInitNotch()` 以新頻率重新設定雙二階陷波係數 |
| ⑨ 陀螺訊號通過更新後的陷波器 | Core 0 | 2 000 Hz | 振動峰值被精準消除後送往 Madgwick AHRS 與 PID |

---

### A.4 程式碼路徑（`fft_cmsis.cpp` + `control core 1 loop`）

```cpp
// Core 1 迴路內（每次執行）
IMURaw imu = bmi088GetLatest();
fftProAddSample(imu.gx * RAD_TO_DEG);   // 加入緩衝區

// 緩衝區滿時（每 ~128 ms）fftProAddSample 內部自動觸發：
//   arm_rfft_fast_f32() → 找峰值箱 → 更新 _peakHz

// Core 1 另一計時（每 50 ms）
if (fftProResultReady()) {
    float peakHz = fftProGetPeakHz();                      // 讀取峰值頻率
    uint16_t eRPM = peakHz * 60.0f / MOTOR_POLE_PAIRS;   // 換算為電氣 RPM
    rpmFilterUpdateRPM(0, eRPM);                           // 更新 Core 0 陷波器
}
```

### A.5 為何 ~8 Hz 更新速率已足夠？

馬達 RPM 的變化速度受限於電調（ESC）的加速時間與螺旋槳的轉動慣量。即使是激進的油門操作，從靜止到最高轉速通常需要 **50–200 ms**。

以最快的 50 ms 加速為例：

```
陷波更新週期 = 128 ms
馬達加速時間 = 50 ms
```

在馬達完成加速的過程中，陷波濾波器最多只會落後 **1–2 個更新週期（128–256 ms）**，在此期間振動抑制效果略有下降，但不會對飛行穩定性造成可見影響。對於一般油門操作（變化速度更慢），陷波濾波器的追蹤幾乎是即時的。

相比之下，Betaflight 在 F7 處理器（480 MHz）上以 200–500 Hz 更新動態陷波，但其方法使用頻率更高的 Biquad 分析而非完整 FFT。本韌體選擇 256 點 FFT 是因為頻率解析度（7.8 Hz/bin）比 Betaflight 的方法更精準，且 CMSIS-DSP 在 Cortex-M33 上的效率足以讓整個運算只佔用 Core 1 約 **0.2 %** 的資源。
